# 특허명세서 최종 점검 프롬프트
**발명명:** 신제품 매출 예측을 위한 다중 에이전트 앙상블 시스템 및 방법  
**명세서 버전:** v2.1  
**적용 심사기준:** 한국 특허법 제42조 / KIPO AI 심사실무가이드 3차 개정(2026.1.15)  
**작성일:** 2026년 4월

---

## 사용 방법

이 프롬프트는 다음 3가지 용도로 활용할 수 있습니다.

1. **변리사 최종 검토 의뢰 시** — 점검 항목 체크리스트로 제출
2. **AI(Claude 등) 점검 요청 시** — 명세서 첨부 후 이 프롬프트 전달
3. **자체 내부 검토 시** — 담당자가 직접 체크하는 항목표로 활용

---

## 지시문

아래에 첨부된 특허명세서를 읽고, 5개 레이어의 점검 항목 각각에 대해 다음 형식으로 평가하시오.

**평가 형식:**
- 상태: ✅ 합치 / ⚠️ 보완 필요 / ❌ 미충족
- 근거: (명세서 내 해당 위치 또는 부재 이유)
- 권고: (보완이 필요한 경우 구체적 수정 방향)

## 레이어 1. 법적 기재요건 (특허법 제42조)

### [1-1] 발명의 명칭

### [1-2] 기술 분야 (제2절)

- [ ] 기술 분야 기술이 청구항의 발명 범위와 일치하는가?
- [ ] 이종성(heterogeneity) 판단 기준 3항이 구체적으로 정의되어 있는가?

### [1-3] 배경기술 (제3절)
- [ ] 선행기술의 한계가 청구항이 해결하는 과제와 직접 연결되는가?
- [ ] 수동 파라미터 입력 방식의 3가지 구조적 한계가 구체적으로 기술되는가?
- [ ] 선행특허 4건(P1~P4) 각각의 한계가 본 발명과 대비되어 명시되는가?
- [ ] 학술 문헌 인용이 서지정보(저자·연도·저널·권호·페이지)를 정확하게 기재하는가?
- [ ] 할루시네이션이 의심되는 서지정보가 없는가? (특히 refPara 8·9·10·12·19)

### [1-4] 발명이 해결하고자 하는 과제 (제4절)
- [ ] 해결하려는 기술적 과제가 배경기술의 한계와 논리적으로 연결되는가?
- [ ] 3가지 기술 과제(파라미터 자동 추정·이종 앙상블 편향 감소·실측 없는 신뢰도 검증)가 각각 청구항 1·2·3에 대응하는가?

### [1-5] 과제 해결 수단 (제5절)
- [ ] 5.1~5.3이 청구항 1·2·3에 각각 대응하는가?
- [ ] 해결 수단이 청구항의 구성과 일치하는가 (청구항에 없는 구성이 여기에만 있지 않은가)?

### [1-6] 발명의 효과 (제6절)
- [ ] 효과가 구체적·수치적으로 기술되어 있는가?
- [ ] "신속한 처리", "정확도 향상", "신뢰도 향상" 등 당연한 효과 표현이 제거되었는가?
- [ ] 각 효과가 해당 청구항의 구성에서 비롯됨을 추론할 수 있는가?
- [ ] 비교 실험 수치(단일 Bass backcasting_score=0.51 vs 이종 앙상블=0.71)가 기재되었는가?

### [1-7] 발명의 상세한 설명 (제8절) — 실시가능성
- [ ] 통상의 기술자가 명세서만 읽고 청구항의 발명을 재현할 수 있는가?
- [ ] LLM 활용 방식(범주·파인튜닝 여부·프롬프트 구조·웹크롤링 방식)이 기재되었는가?
- [ ] 확산 패턴 유형 4종(급성장형·S자형·완만형·네트워크 가속형)별 파라미터 탐색 공간이 구체적 수치로 기재되었는가?
- [ ] 파라미터 DB 구성 요소·갱신 주기·최소 사례 수가 기재되었는가?
- [ ] 수식·코드 블록에 정의되지 않은 변수가 없는가?
- [ ] 실시예 2개 이상이 청구항의 구성을 실제로 포함하여 기술되었는가?

### [1-8] 요약서 (제10절)
- [ ] 200자 내외로 간결하게 작성되었는가?
- [ ] 발명의 기술적 핵심(LLM 패턴 식별·이종 앙상블·IP 강도 통합·사전 신뢰도 검증)을 포함하는가?
- [ ] 대표 도면 번호가 지정되었는가?
