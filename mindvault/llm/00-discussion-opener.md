## 🎯 우리가 답해야 하는 5개의 근본 질문

이 프로젝트의 AI 모델 전략을 "제대로" 설계하려면, **분석 이전에 전략적 제약조건**을 먼저 확정해야 합니다. 제약이 다르면 최적해가 완전히 달라집니다.

### 질문 A — 운영 환경·예산
1. **월간 LLM API 비용 한도**는? (예: $500 / $5,000 / $50,000)
2. **자체 GPU 서버(A100·H100)** 운영 가능한 예산이 있는가?
3. **초기 MVP는 API-Only로 시작 → 자체호스팅 후속 도입** 수용 가능한가?

### 질문 B — 사용자 및 데이터 규모
1. 예상 **연구자 수 / 일일 일지 건수**는? (소규모 연구실 vs 대학 단위 vs 국책사업단)
2. **학습 데이터 확보** — 실제 연구자 일지를 라벨링 대상으로 확보 가능한가? 몇 건 확보 가능?
3. 한국어 외 **영어/중국어** 일지도 처리해야 하는가?

### 질문 C — 보안 등급 현실
1. 대상 R&D 프로젝트의 **보안 등급 분포**는 어떠한가?
   - 공개 ○○% / 주의 ○○% / 보호 ○○% / 비밀 ○○%
2. **보호/비밀 등급**이 유의미한 비중이면 자체호스팅 필수, 아니면 API-Only로 비용 절감 가능
3. **정부 기관 보안 감사 대상**인가? (NIS, 과기정통부 등)

### 질문 D — 팀 역량
1. 팀에 **ML 엔지니어** 또는 **LLM Fine-tuning 경험자**가 있는가?
2. **한국어 NLP**(KoBERT, Kiwi, Mecab) 경험 보유자는?
3. MLOps(모델 배포·모니터링) 운영 가능 인력은?

### 질문 E — 핵심 성능 목표
1. **일지 파싱 지연시간** 목표 — 실시간(<2초) vs 배치(야간 일괄)?
2. **Hallucination 제로** — 어느 수준까지 엄격한가? (특허 CP3 기준인 "수치 판정 AI 배제"가 유일한 선? 아니면 모든 factual claim에 citation 의무?)
3. **합격 기준 정확도** — 파싱 80% vs 90% vs 95%?

## 📋 논의 프레임워크 (Discussion Framework)

질문 A~E에 답이 나오면, 다음 6개 레이어를 **순차로** 설계 결정합니다.

```
┌─────────────────────────────────────────────────────────────┐
│ L1. 모델 포트폴리오 (Model Portfolio)                        │
│    어떤 모델들을 조합할 것인가?                               │
│    Opus / Sonnet / Haiku / ExaOne / Llama / Qwen / KoBERT 등 │
├─────────────────────────────────────────────────────────────┤
│ L2. 작업-모델 매핑 (Task-Model Matrix)                       │
│    10개 AI 작업을 어떤 모델에 배정할 것인가?                  │
│    (파싱/분류/추출/생성/추론/합성/검색/대화/탐지/수치판정)    │
├─────────────────────────────────────────────────────────────┤
│ L3. 특화 방법 (Specialization Strategy)                      │
│    RAG / Few-shot / LoRA FT / Full FT / Tool-use 어떻게 조합? │
├─────────────────────────────────────────────────────────────┤
│ L4. 한국 R&D 파이프라인                                       │
│    형태소 분석 → NER → Disambiguation → LLM Fallback          │
├─────────────────────────────────────────────────────────────┤
│ L5. 보안 등급별 라우팅                                        │
│    공개/주의 → API. 보호/비밀 → 자체호스팅                    │
├─────────────────────────────────────────────────────────────┤
│ L6. Hallucination 방어 아키텍처                               │
│    Structured Output + Confidence + Rule Engine + PROV-O     │
└─────────────────────────────────────────────────────────────┘
```

## 🧭 논의 진행 방식

**Step 1** — 질문 A~E에 대해 사용자가 답변 (또는 "모른다/유연하다" 표시)
**Step 2** — 답변을 기반으로 제약조건 고정
**Step 3** — L1~L6을 순차 설계 (각 레이어마다 대안 3~5개 비교 → 사용자 승인)
**Step 4** — 완료된 결정을 `mindvault/decisions/002-ai-model-architecture.md`로 공식화
**Step 5** — 실행 계획 수립 (학습 데이터 수집 / 모델 선정 / 파이프라인 구축)
