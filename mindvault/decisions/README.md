# Decisions - 의사결정 기록 & 근거

## 📋 Architecture Decision Records (ADR)

이 섹션은 모든 주요 의사결정을 기록하며, 각 결정은:
- **Status**: Proposed / Accepted / Deprecated / Superseded
- **Context**: 의사결정이 필요한 배경
- **Decision**: 선택한 결정
- **Consequences**: 예상되는 결과
- **Alternatives**: 검토한 다른 선택지

## 📋 Decision Categories

| Category | Purpose |
|----------|---------|
| [ADR-0001~0050](./adr/) | 아키텍처 의사결정 |
| [Model Selection](./llm-decisions/) | LLM 모델 선정 기록 |
| [Feature Priority](./feature-decisions/) | 기능 우선순위 의사결정 |
| [Trade-offs](./trade-offs/) | 기술적 트레이드오프 분석 |

## 🔍 How to Record Decisions

```markdown
# ADR-XXXX: [Title]

**Date:** YYYY-MM-DD  
**Status:** Proposed | Accepted | Deprecated | Superseded  
**Decision Maker:** [Name]

## Context
[배경 및 필요성]

## Decision
[선택한 방안]

## Reasoning
[선택 이유]

## Alternatives Considered
[검토한 다른 선택지]

## Consequences
- Positive: [...]
- Negative: [...]

## References
[관련 자료 링크]
```
