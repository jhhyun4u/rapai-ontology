## 🔄 데이터 흐름 (Data Flow)

```
User Input (연구계획서)
    │
    ├─→ [Ontology Layer] 프로젝트 유형 식별
    │   └─→ PMBOK 매핑, NPD 선택
    │
    ├─→ [Decision Engine] 의사결정 실행
    │   ├─→ WBS 생성 (근거 기록)
    │   ├─→ Schedule 수립 (제약 고려)
    │   ├─→ 리소스 계획 (가용성 고려)
    │   ├─→ 리스크 분석 (식별→평가→대응)
    │   └─→ KPI 정의 (측정 가능한 목표)
    │
    ├─→ [Knowledge Repository] 저장
    │   ├─→ 의사결정 기록 (근거 포함)
    │   ├─→ 프로젝트 계획 문서
    │   └─→ 가정 & 제약 명시
    │
    ├─→ [Tracing Layer] 기록
    │   ├─→ 모든 선택지 추적
    │   ├─→ 의사결정 체인 기록
    │   └─→ 근거 링크
    │
    └─→ [Output] 사용자 제시
        ├─→ 종합 계획서
        ├─→ 대시보드
        └─→ 실행 계획
```

## 🏛️ 레이어별 책임 (Layered Responsibilities)

### Ontology Layer
```
입력: 연구계획서 원문
처리: 도메인 개념 매핑
출력: 분류된 프로젝트 타입 + 적용 프레임워크
```

### Decision Engine Layer
```
입력: 분류된 프로젝트 + 제약사항
처리: PMBOK/NPD 기반 의사결정
출력: WBS, Schedule, 리소스, 리스크, KPI
근거: 모든 선택지 기록
```

### Knowledge Repository Layer
```
입력: 의사결정 결과
처리: 구조화된 저장 + 상호 참조
출력: 프로젝트 지식베이스
유지: 정기적 갱신 + 자동 검증
```

### Tracing & Observability Layer
```
입력: 의사결정 + 모니터링 데이터
처리: 근거 추적 + 실시간 모니터링
출력: 결정 로그 + 상태 대시보드
검증: Hallucination 탐지
```

## 🔐 아키텍처 불변조건 (Invariants)

✅ **반드시 유지해야 할 조건**

1. **분리된 계층** (Separated Layers)
   - 각 계층은 그 아래 계층만 의존
   - 교차 의존성 금지
   - 기계적 강제

2. **근거 기록** (Evidence Linking)
   - 모든 주요 의사결정에 근거 기록
   - 선택지와 이유 명시
   - 가정 기록

3. **자동 검증** (Mechanical Validation)
   - Lint 규칙
   - 구조적 테스트
   - CI 게이트

4. **버전 관리** (VCS Everything)
   - 모든 아티팩트 Git 관리
   - 변경 이력 추적
   - 감사 증적

5. **자동 정리** (Garbage Collection)
   - 정기적 드리프트 감지
   - 자동 수정 제안
   - 일관성 유지
